About this site

How this works

This isn't a polished tutorial site — it's a live learning log. Every module was built during an actual study session, working through the concept from scratch against real pipeline code.

Each interactive artifact is designed to be a reference you return to, not a one-time read. Decision cards and walkthroughs are tools for writing pipeline code — not just reading summaries.

The site grows one session at a time. When a module is done, it appears here. No publish pipeline — just open and explore.

  • ⚡
    Mechanism firstEvery concept is derived from examples. If you can't trace a record through the pipeline, you don't know the concept yet.
  • 🎯
    Use depth, not recognitionThe goal is being able to apply the concept directly in code — not just recognise it in a quiz.
  • 🔁
    Built on the jobEvery track connects to a real system. No toy examples that don't transfer.
