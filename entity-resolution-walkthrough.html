Stage 2 — Standardize
StandardizationPlugin normalizes fields: lowercase email, strip punctuation from name, E.164 phone. This is what your YAML plugins do before blocking.
df = StandardizationPlugin(config).transform(df)

Stage 3 — Block
Assign each record to one or more blocks. Only records sharing a block key will be paired. This avoids the O(n²) full cross-join.
df = df.withColumn("block_key", soundex("name_std"))

R1+R2 share soundex J525 even though "Smith" ≠ "Smyth" — that's soundex doing its job. R2 also gets no phone block (null), so it only has one path to find its duplicate.
Stage 4 — Shuffle & Self-Join
Records with the same block key shuffle to the same partition (desk). The self-join generates candidate pairs within each partition — only records on the same desk meet.
pairs = df.alias("a").join(df.alias("b"), (a.block==b.block) & (a.id < b.id))

Stage 5 — Score & Classify
Each pair gets a similarity score. Jaro-Winkler for names, exact match for phone/email, weighted average → final score → verdict.
pairs = ScoringPlugin(config).transform(pairs)
